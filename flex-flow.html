ทำไมต้อง flex-flow? — 2 บรรทัด → 1 บรรทัด

เพราะ flex-direction กับ flex-wrap เนี่ย แทบจะไปด้วยกันเสมอ เขียนแยก 2 บรรทัดก็ได้ผลเหมือนกัน แต่เสียเวลา — เขียนเป็น flex-flow บรรทัดเดียวจบ ผลลัพธ์เหมือนกันเปี๊ยบ

แบบเก่า · 2 บรรทัด
.container { display: flex; flex-direction: row; flex-wrap: wrap; }
1234
แบบ shorthand · 1 บรรทัด ✓
.container { display: flex; flex-flow: row wrap; }
1234
💡 กฎเดียวที่ต้องจำ: flex-flow = flex-direction + flex-wrap เรียงติดกันในบรรทัดเดียว — ส่วนจะใส่ direction หรือ wrap ก่อนนั้น ไม่สำคัญ (เดี๋ยวพูดในโซน 04)

04 ·

ลำดับค่า ไม่สำคัญ 🔁

ข้อดีอย่างหนึ่งของ shorthand คือ browser ฉลาดพอที่จะแยกแยะได้ว่าค่าไหนคือ direction อันไหนคือ wrap — ดังนั้น flex-flow: row wrap กับ flex-flow: wrap row ให้ผลเหมือนกัน

ลองสลับ:
✨ แต่ convention จริง: นิยมเขียน direction ก่อน wrap เพราะอ่านแล้วเข้าใจง่ายกว่า (คนคุ้นกับ "ทิศทาง → การแบ่งบรรทัด") แต่เขียนสลับก็ใช้ได้เสมอ
05 ·

จุดที่คน พลาดบ่อย ⚠️

① ลืมว่าเป็น shorthand — ทำให้ทับค่าเก่า

เพราะ flex-flow คือ shorthand ของทั้งสอง property ถ้าเขียนทีหลัง มันจะรีเซ็ตค่าที่ไม่ได้ระบุเป็น default หมด

/* อันนี้จะพัง — wrap ถูกรีเซ็ตเป็น nowrap */
flex-wrap: wrap;
flex-flow: row; /* ← ทำให้ wrap กลายเป็น nowrap */

② ถ้าใส่ค่าเดียว อีกค่าจะเป็น default

เขียน flex-flow: row อย่างเดียว — browser จะตั้ง flex-wrap เป็น nowrap ให้อัตโนมัติ (เหมือนไม่ได้เขียน) ใช้ค่าเดียวได้ แต่ต้องรู้ผลลัพธ์

/* ทั้งสองบรรทัดให้ผลเหมือนกัน */
flex-flow: column;
flex-flow: column nowrap;

③ order ของค่าไม่สำคัญ แต่ direction ก่อนเข้าใจง่ายกว่า

จริงๆ แล้ว flex-flow: row wrap กับ flex-flow: wrap row ให้ผลเหมือนกัน แต่เขียน direction ก่อนเป็น convention ที่คนอ่านโค้ดเข้าใจง่ายกว่า — เว้นแต่ในทีมมีกฎอื่น ก็ใช้ตามนั้น

/* ทั้งสองบรรทัดให้ผลเหมือนกัน ✓ */
flex-flow: row wrap;  ← แนะนำ
flex-flow: wrap row;

Cheatsheet 30 วินาที 📋

ตารางค่าผสม flex-flow — ทั้ง 12 ค่า

แถว = flex-direction · คอลัมน์ = flex-wrap · ★ = ค่าที่ใช้บ่อย

nowrap
wrap ★
wrap-reverse

จำไว้ 3 ข้อ:

  1. flex-flow = flex-direction + flex-wrap ในบรรทัดเดียว
  2. เขียนค่าเดียวได้ — อีกค่าเป็น default (nowrap)
  3. order ไม่สำคัญ — แต่นิยม direction ก่อน wrap
📚 อ่านเพิ่มเติม: ดูรายละเอียดแต่ละ property ได้ที่ flex-direction และ flex-wrap — หรือลองตัวถัดไป justify-content
